**src/middleware/domain_context.py:**

Create a FastAPI middleware (or dependency) that injects RLS context into the database session.

Approach: Use a FastAPI dependency (not ASGI middleware) for cleaner integration with the session lifecycle.

```python
from fastapi import Request, Depends
from sqlalchemy.ext.asyncio import AsyncSession
from sqlalchemy import text
from src.core.database import get_db

async def set_domain_context(
    request: Request,
    session: AsyncSession = Depends(get_db),
) -> AsyncSession:
    """
    Sets the PostgreSQL session variable for RLS domain isolation.

    Domain ID comes from:
    1. X-Domain-ID header (set by frontend after domain selection)
    2. Falls through to None (no domain context — global queries)

    Uses SET LOCAL which is transaction-scoped and automatically
    cleared when the transaction ends. Cannot leak via connection pool.
    """
    domain_id = request.headers.get("X-Domain-ID")
    if domain_id:
        # Validate UUID format to prevent injection
        try:
            uuid.UUID(domain_id)  # Raises ValueError if invalid
        except ValueError:
            raise HTTPException(400, "Invalid domain ID format")
        await session.execute(
            text("SET LOCAL app.current_domain_id = :domain_id"),
            {"domain_id": domain_id},
        )
    else:
        # Reset to empty — RLS policies should handle NULL/empty gracefully
        await session.execute(
            text("SET LOCAL app.current_domain_id = ''")
        )
    return session
```

CRITICAL: Use parameterized query (`:domain_id` with bind) NOT string format. Validate UUID format before using. `SET LOCAL` with named parameter via SQLAlchemy `text()`.

NOTE on SET LOCAL with parameters: PostgreSQL SET LOCAL does not support $1 bind parameters. Use this safe pattern instead:
```python
# Validate UUID format first (done above), then use literal_column or format safely
await session.execute(text(f"SET LOCAL app.current_domain_id = '{domain_id}'"))
```
Since we validate UUID format strictly before interpolation, this is safe from injection. Document this pattern clearly.

```python
import uuid
from datetime import datetime
from sqlalchemy.ext.asyncio import AsyncSession
from src.models.event import Event

async def log_event(
    session: AsyncSession,
    *,
    actor_id: uuid.UUID | None = None,
    actor_type: str,  # 'user' | 'agent' | 'system'
    action: str,
    resource_type: str | None = None,
    resource_id: uuid.UUID | None = None,
    domain_id: uuid.UUID | None = None,
    payload: dict | None = None,
    ip_address: str | None = None,
) -> Event:
    """
    Append an event to the audit log.

    This is the ONLY way to create events. The events table
    is append-only: no UPDATE or DELETE is granted to app_user.
    """
    event = Event(
        actor_id=actor_id,
        actor_type=actor_type,
        action=action,
        resource_type=resource_type,
        resource_id=resource_id,
        domain_id=domain_id,
        payload=payload,
        ip_address=ip_address,
    )
    session.add(event)
    await session.flush()
    return event
```

IMPORTANT: The `log_event` function takes explicit kwargs, not **kwargs. This prevents passing arbitrary data to the model.
